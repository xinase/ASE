## 6.1 敏捷的流程简介

敏捷是一系列价值观和方法论，旨在应对变化、快速交付价值。2001年的敏捷宣言强调个人和交流、可用的软件、与客户合作、响应变化。这些原则在AI时代依然成立，但被赋予了新的内涵：AI使频繁交付和快速响应变化成为可能，同时也扩大了“变化”的范围（数据漂移、模型局限、伦理考量等）。

## 6.2 敏捷流程的问题和解法

敏捷在实践中常面临依赖关系复杂、任务分解粗粒度、忙闲不均、中途需求变更、燃尽图失真、完成定义模糊等问题。实战解法包括：依赖可视化、任务拆到“一人一天”、公开认领+召集人协调、严格的“完成的定义”、剩余工时燃尽图等。

## 6.3 敏捷的团队与哲学

敏捷流程的效能取决于团队本身。敏捷不是万能药，它更像一面镜子，能放大强大团队的优势，同时暴露薄弱团队的缺陷。敏捷哲学强调以人为本、技术卓越、极简主义，这些原则在AI时代更为重要。

## 6.4 AI时代的敏捷流程-BMad方法

传统Scrum的固定时间盒在AI项目中面临挑战。BMad（Benchmark→Model→Apply→Deploy）方法将Sprint目标重塑为“完成一次完整价值验证”，通过明确基准、模型训练、应用集成和部署监控，实现真正的数据驱动迭代，适合AI开发的不确定性。

## 6.6 开源与敏捷：协作哲学的冲突与融合

敏捷强调面对面同步交流，开源强调异步文字协作，二者在沟通哲学上存在根本差异。但在AI时代，工具（如GitHub、远程协作平台）帮助二者走向融合，形成异步优先、远程友好、高效协作的现代模式。

## 6.7 案例分析-超算项目之垂直切片
