# 第10章　典型用户和场景

要完成的任务本身就包含了场景，而场景（context）才是关键。  —克莱顿·克里斯坦森，创新专家

## 10.1 典型用户和典型场景

典型用户（Persona）是基于真实用户调研构建的虚拟人物，用于代表核心用户群体的特征、动机和痛点。结合Job To Be Done（JTBD）理论，能帮助团队从“用户需要什么功能”转向“用户想完成什么任务”。在AI时代，Persona的定义、验证和迭代都可被AI大幅加速，同时必须新增“AI安全威胁用户”（如数据投毒者、提示词注入者）等新型典型用户。

## 10.2 用例（Use Case）

用例是结构化的场景描述工具，包括标题、角色、主要成功场景、步骤和扩展场景。它与Persona、场景共同构成需求分析的桥梁。用例图可直观展示系统边界与交互。在AI时代，用例可由AI辅助生成，并增加AI特有扩展场景（如模型幻觉、提示词注入）。

## 10.3 功能说明书和技术说明书

功能说明书（Functional Spec）面向非技术角色，回答“做什么”；技术说明书（Technical Spec）面向工程师，回答“怎么做”并记录决策权衡。两者共同构成活文档，在AI时代可借助AI生成初稿、自动检查一致性，并支持Docs-as-Code。

## 10.4 AI赋能的功能驱动的设计（FDD）

FDD通过领域建模、功能列表、计划、设计、实现五步实现价值交付。AI将其全面增强：AI充当领域知识外脑、自动生成模型/功能列表、推荐设计、加速代码与测试，使FDD在陌生领域和高速迭代中更高效。

## 10.5 从功能定义到产品演进

Cynefin框架与确定性-变更成本矩阵帮助团队避免系统性误判，实现“以最低成本快速验证是否足够好”。核心是构建自适应反馈飞轮：提出核心假设 → 选择最经济验证方法 → 执行 → 分析学习 → 调整认知。

### 1. AI时代典型用户构建与验证  
**这针对于10.1小节**（典型用户和典型场景）。

为一个AI产品（例如AI个人知识管理助手或智能批改系统）定义至少两个典型用户（含新型“AI安全威胁用户”）。使用AI工具辅助构建Persona模板，并设计一个低成本验证实验（人肉MVP或A/B测试）。说明如何迭代Persona并处理“被删除的典型用户”。

### 2. 用例在AI功能中的应用  
**这针对于10.2小节**（用例）。

为“AI批改简答题”或你感兴趣的AI功能编写一个完整用例（含主要成功场景和至少3个扩展场景）。画出对应用例图，并说明AI如何辅助生成/验证用例。讨论AI时代用例的边界条件应新增哪些AI特有场景（如模型幻觉、提示词注入）。

### 3. 功能说明书与技术说明书的AI协同编写  
**这针对于10.3小节**（功能说明书和技术说明书）。

选择一个AI功能，撰写其功能说明书（回答8个核心问题）和技术说明书（包含决策记录）。说明AI如何帮助清除“需求怪味”、生成边界条件列表和TDD测试用例。讨论如何将两者作为“活文档”与代码仓库同步维护。

### 4. AI赋能FDD的实践规划  
**这针对于10.4小节**（AI赋能的功能驱动的设计）。

以一个陌生领域AI项目为例，规划FDD五步的AI增强流程。重点说明AI在“总体模型构建”“功能列表生成”“智能开发计划”中的具体作用，并设计一个“探索Sprint”来验证高风险功能。

### 5. 需求验证框架在AI项目中的应用
