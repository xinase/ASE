## 5.1 非团队和团队

团队与非团队的本质区别在于是否拥有一致的集体目标、明确的分工以及成员间的相互依赖。非团队只是临时聚集在一起完成各自任务的人群，而真正的团队需要共同的目标和协作才能产生大于个体之和的价值。

## 5.2 软件团队的模式

软件团队有多种成熟模式，包括主治医师模式、明星模式、业余剧团模式、秘密团队（Skunk Work）、特工团队（SWAT）、交响乐团模式、爵士乐模式、功能团队、官僚模式等。每种模式适用于不同阶段、不同规模和不同创新需求的项目。团队拓扑理论进一步指出，应以期望的系统架构为目标，反向设计合适的团队结构。

## 5.3 开发流程

开发流程是团队协作的骨架。从最早的“写了再改”模式，到瀑布模型及其各种变形、统一流程（RUP）、老板驱动流程、渐进交付（MVP）、DevOps，再到AI时代的AIOps，每种流程都反映了当时的技术约束和组织现实。所有模型都有局限，但某些模型在特定场景下非常有用。

## 5.4 开源浪潮中的团队和流程

开源是当今软件开发的主流形式。其历史根源可追溯到早期共享代码的实用主义，后经黑客文化、意识形态冲突与商业融合，最终形成全球分布式、精英治理的独特协作模式。开源项目的团队模式与开发流程高度依赖社区贡献、透明沟通和“早发布、常发布”原则。初学者参与开源是检验和提升工程能力的最佳实践。
