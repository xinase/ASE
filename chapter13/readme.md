## 13.1 由核事故引出的基本名词解释

1986年切尔诺贝利核事故中，辐射检测仪指针卡死在最大量程3.6伦琴/时，管理人员误判为“可控”。这一事件深刻揭示了软件测试的核心：我们不仅要发现目标系统的Bug，更要确保监测工具本身可靠。测试的根本目标是回答V&V两个问题——Verification（我们是否正确地构建产品？）与Validation（我们是否构建了正确的产品？）。在AI时代，这一目标被扩展为同时验证代码逻辑、模型行为、数据质量与服务稳定性。

## 13.2 AI系统的bug和test

AI系统的Bug与传统软件不同，其根源往往不在代码逻辑，而在数据偏见、模型局限或推理服务不稳定。AI Bug需要新的分类（模型Bug、数据Bug、推理服务Bug、AI偏见Bug）和测试策略。测试AI不再是一次性活动，而是嵌入MLOps生命周期的持续过程。
