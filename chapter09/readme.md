## 9.1 PM定义与分类

PM（Project/Program/Product Manager）是软件团队中连接业务、用户与技术的核心角色。其七项核心任务包括愿景转化、生命周期管理、规格维护、利益代表、变更管理、平衡三超压力以及数据驱动改进。在AI时代，PM更需聚焦需求探索、价值验证与战略思考，AI工具成为其强大辅助而非替代者。

## 9.2 PM核心能力与知识体系

PM的核心能力包括知识管理（显性/默会/已知未知）、清除需求怪味、分析决策、影响力与文化构建。团队知识四个象限与新闻稿（Working Backwards）方法是其独特工具，帮助PM挖掘默会知识并构建高效文化。

## 9.3 推动高效的团队讨论

高效会议需有序流程（事实→情绪→乐观→悲观→创意），并建立高信任文化。AI会议总结工具可解放PM，但文化与责任感仍是核心。

## 9.5 PM的挑战与成长

AI不会取代PM，而是放大其对默会知识的洞察力与人文判断。PM需从具体小事中积累隐性知识，成为方向定义者与最终责任人。
