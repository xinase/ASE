# 第17章　人，绩效和职业道德

You manage things, you lead people.  ——Grace Hopper

## 17.1 领导力：从自我生存到引领团队

领导力不是未来被授予的头衔，而是从第一天起就必须展现的生存姿态。格蕾丝·霍珀的名言提醒我们：管理事务（things），带领人（people）。在AI重塑团队规模与速度的今天，领导力的根基是“自我领导力”——能否在日常中展现极致投入与创造“优质资产”的能力。真正的领导力有三种形态：技术领导力、思想领导力与项目领导力，最终指向教练式领导力，让团队成员成为更好的自己。

## 17.2 识人与用人：构建高效团队的基石

识人需评估能力（知识+专业技能+职业技能）与动力（热情+投入+承诺）。四象限人才模型（积极初学者→自立成就者）揭示了个人动态发展轨迹，领导者需成为教练，因材施教，实现能力与动力的乘数效应。

## 17.3 团队发展的生命周期

团队如人，会经历萌芽（奠定规则）、磨合（化解冲突）、规范（形成秩序）、创造（高效自治）四个阶段。领导者需在不同阶段切换角色，从“快刀斩乱麻”到“保护伞”，避免长期停留在“假团队”陷阱。

## 17.4 团队角色与责任

团队成员投入度不同：猪（全身心投入）、鸡（重要贡献但风险可控）、鹦鹉（围观者）。RASCI/DRI模型将投入转化为清晰责任边界，确保“猪”主导关键决策，避免责任真空。

## 17.5 绩效管理的艺术与科学

绩效评估需超越Input Metrics（代码行数、忙碌程度），聚焦Output Metrics（业务价值、资产而非负债）。萝卜（快了不洗泥）与白菜（慢工细活）的辨析提醒我们：真正的英雄是构建优质资产、降低长期风险的人。AI时代，绩效体系必须奖励“白菜”式贡献，避免制造高息负债。

## 17.6 构建顶尖团队环境：几种创新文化的范式

顶尖团队环境有三种模式：培育式（心理安全+相态分离）、牵引式（使命驱动+首席工程师牵引）、生存式（战时DRI+激进透明）。领导者需根据情境切换，避免将战时当平时或平时当战时。关怀型领导力是第三种被低估的力量，通过情感连接激发长期韧性。

## 17.7 软件工程师的职业道德

AI时代，工程师掌握造福或危害社会的巨大力量。IEEE/ACM《软件工程师职业道德规范》是底线，但更重要的是在全流程中主动思考：数据伦理、偏见公平、滥用风险、透明度与用户操纵。道德不是“老板让做的”，而是“思想领导力”——主动提出更好方案、保留反对意见记录、守护用户信任。
