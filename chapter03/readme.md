# 第3章　软件工程师的成长

Code is cheap, show me the prompts.   —Anonymous (2025)

## 3.1 个人能力的衡量与发展

软件工程师的价值不能仅靠“加班时长”来衡量。职业工程师需要可量化的、稳定的产出，包括项目规模、耗时、质量和交付稳定性。在AI时代，工程师还需要考虑AI资源消耗与可重用性。

## 3.2 个人开发流程——PSP

个人软件过程（PSP）帮助工程师定义过程、度量过程、分析数据并持续改进。在AI时代，PSP被升级为“问题分解→AI辅助实现→验证→复盘”的循环，AI接管了大量低层次编码工作，但核心循环仍由人类主导。

## 3.3 技能的反面

技能的反面是“解决问题”。真正的高阶技能是将低层次问题转化为肌肉记忆，从而把认知资源释放到更高层次的分析、评价和创造。AI时代，这一理论被进一步强化：AI接管了90%的低层次劳动，工程师必须专注剩余10%的高阶能力。

## 3.4 软件工程师的思维误区

工程师常陷入分析麻痹、追牦牛、过早优化、过早泛化等思维误区。AI时代，这些误区被放大：AI生成的大量方案可能导致选择困难或过度依赖。

## 3.5 从知道到创造的认知飞跃

AI让“能指”（代码）与“所指”（意图）的界限变得模糊。工程师必须完成从记忆→理解→应用→分析→评价→创造的认知飞跃。AI是能力显影剂，把高阶认知能力推到前台。

## 3.6 软件工程师的职业发展

AI时代工程师的职业发展从“单人乐队”转向“指挥家”。价值体现在设计意图、系统治理和业务洞察上。职业等级不再仅看代码量，而是看稳定交付、影响力与AI治理能力。

## 2026年附录：本章思考与练习

（本附录专门对应第3章内容，帮助学生将个人能力衡量、思维误区、认知飞跃和职业发展与AI时代的真实实践相结合。建议先通读全章后再开始练习。）

### 1. 选哪一种“AI时代的工程师”？  
**这针对于3.1和3.6小节**（个人能力衡量与职业发展）。

作为一个2026年的软件工程师，你觉得自己表现如何？如果让你选择以下四种角色，你最想成为哪一种？为什么？

a) 完全依赖AI生成代码，快速交付，但对AI输出几乎不做验证。  
b) 喜欢用AI尝试最新技术，经常在项目中引入前沿AI方案，但经常导致交付延误。  
c) 已经处理过大量AI辅助项目，能稳定交付高质量代码，同时清楚AI的局限性。  
d) 拒绝使用AI，坚持“纯手工”编码，认为这样才能保持“真正”的能力。

**讨论**：① 你要选哪一种？  
② 在AI时代，软件工程师是否还需要正式的职业证书或资格考试？为什么？
