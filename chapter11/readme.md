## 11.1 分析和设计方法

## 11.3 自动生成代码的历史

自动编程的探索从1950s的符号主义（FORTRAN、COBOL）开始，历经1970-90s的CASE工具、1980-90s的“程序员学徒”、1990-2000s的意图编程，直至2010s至今的统计学习与LLM范式。历史表明，每一次浪潮都未能完全取代人类智慧，而是将抽象层次不断提升。

## 11.5 AI时代：软件工程师的成长与未来

AI擅长语法与模式生成，但缺乏语义理解和默会知识。工程师的核心价值在于架构决策、问题定义、系统抽象与遗留系统维护。AI是增强智能的放大器，而非替代者。工程师需聚焦“做什么”和“为什么”，构建驾驭AI的“脚手架”。

## 11.7 现代软件工程的“脚手架”：源代码管理

源代码管理是CI/CD的基础，确保团队在AI生成代码速度指数级增长的时代仍能可靠协作、版本追溯和质量控制。现代工具需支持多语言、分支管理、自动化门禁和历史追溯。

## 11.8 每日构建到持续集成

每日构建（Daily Build）演进为现代CI/CD体系。团队需通过构建大师机制、Bug红线（小强地狱）等文化实践，确保AI时代高频代码生成下的质量底线。Code Complete是里程碑，标志着从“创造”进入“打磨”阶段。
